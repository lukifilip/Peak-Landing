# App Context — Full Overview

## What the App Is

This is a Flutter/Dart mobile application that combines two core pillars:

1. **A skill-learning platform** — Users learn digital skills through AI-generated, curated learning paths. The goal is to offer the best possible skill learning experience available, with real-world applicability at the center of every decision.
2. **A professional social network** — Users can create projects, collaborate, find talent, hire people, apply for open roles at companies, and build a verifiable public portfolio of their skills and completed work.

These two pillars are deeply interconnected. Skills learned and verified within the app become a credentialing system that companies and individuals can trust when hiring or collaborating.

---

## Core Philosophy

- Skills are earned, not self-reported. A user's skill level within the app is based on actual completed work (tasks, exercises, projects), not just claims.
- The platform is built on user integrity. Anti-cheat systems are not a current priority. The projects and tasks are designed to be hard enough that genuine learning is required to pass, and companies will quickly identify users who have cheated when they start working together.
- The focus is on digital skills that are in high demand and have real-world applicability.
- Quality over quantity — every piece of content (sources, projects, tasks) passes through multi-metric filters before being included in any learning path.

## Skill Learning System

### How Skill Learning Plans Are Created

The skill learning plans are AI-generated but heavily curated through a multi-step pipeline:

1. **Scraping** — Relevant information about each skill is scraped from the internet using the Gemini API (latest version). Sources are evaluated against several quality metrics. A source must score above 7/10 on quality to be included. Other metrics include relevancy, real-world applicability, and recency.
2. **Structuring** — Claude Sonnet 4.6 and Claude Opus 4.6 are used to structure the skill learning plans and create all documentation, explanations, and written content within them.
3. **Visuals** — Gemini's lightweight/nano model is used to generate relevant images for the skill learning plans. Graphs and statistics displayed within the learning plans are generated by Claude Sonnet 4.6 and Claude Opus 4.6.
4. **Projects** — Projects used within the skill learning plans are sourced from public repositories (e.g., GitHub) and other public sources. These are real projects that real users have built. They go through the same multi-metric filtering pipeline to ensure quality, relevancy, and real-world use.

### How Learning Paths Work

Each skill has a structured learning path that progresses through multiple difficulty stages (e.g., Novice → Beginner → Intermediate → Advanced → Expert). At each stage:

- The user **absorbs information** (reading, watching, studying structured content).
- The user **completes tasks and exercises** to implement what they've learned.
- The user **builds and submits a project** to demonstrate competency and unlock the next stage.

As users progress, they accumulate **skill points** that represent verified competency at a specific level for that skill.

### Credential Value

A user's skill level and points are visible on their public profile. This acts as a verifiable credential — companies and individuals can look at a user's profile and know with reasonable confidence what skills they have and at what level, based on the projects they have completed within the app.

Early stages (Novice/Beginner) are easier and more AI-solvable. As the user progresses, projects become significantly harder, more practical, and more demanding. The credential becomes more meaningful at higher skill levels.

## App Structure — Main Navigation

The app has several main tabs/pages accessible from the bottom navigation bar (or equivalent main navigation):

1. **Home** — Displays available skills. Users can browse and start learning any skill. Search button (top right) allows searching for specific skills.
2. **Discover** — Displays recommended companies and open roles. Users can apply for roles, explore companies, and create their own company. Search button (top right) allows searching for companies or open roles.
3. **Search** *(currently being redesigned — see To-Do file)* — Currently a page for finding other users by username/name. Being redesigned into a unified search experience.
4. **Chats** — Displays all private chats and group/project chats the user is part of.
5. **Profile** — The user's own profile, showing their skills, skill levels, portfolio, completed projects, and social info.

## Social Features

### User Profiles
Each user has a public profile displaying:
- Name, username, profile picture
- Skills they are learning and have completed, with skill levels
- Portfolio of completed projects (from within the app and from the company section)
- Work history / company affiliations

### Messaging
- **Private (DM) chats** — A user can DM another user by visiting their profile and tapping the message button. The chat is then saved in the Chats tab.
- **Project/Group chats** — When a project is created inside the company section and members are assigned to it, a group chat is automatically created with all assigned members. The chat is named after the project title. This removes the friction of manually creating group chats.
- All chats (private and group) are accessible from the main Chats tab.

### Discover Page
- Displays recommended open roles and companies to the user based on their skills and skill levels.
- Users can apply for open roles directly from this page.
- Users can browse companies and view their public profiles.
- Users can create their own company from this page.

## Company Section

### Overview
Organizations can register as companies within the app. The company section is currently free and will be monetized in the future. Companies can manage employees, create and assign projects, open public hiring roles, and communicate through project chats.

### Roles
There are three roles within a company:

- **Employee** — Default role when joining a company. Can view and work on assigned projects, communicate with team members, and submit completed work.
- **Manager** — Promoted from employee by an officer or manager. Can create and assign projects, create open hiring roles, and view company-wide data (projects, people, open roles). Cannot see financial data.
- **Officer** — The company creator is automatically an officer. Has all manager permissions plus access to financial insights (revenue, costs, payouts). Can privately message any member and manage managers.

### Main Pages

#### Company Home (`company_home.dart`)
- **CTA Section** — Role-dependent call-to-action (e.g., "Complete Project" for an employee).
- **Our Team** — Total member count and currently active/online members.
- **Our Mission** — Company mission statement.
- **Assigned Projects** *(conditional)* — Shows unconfirmed project assignments for the current user. Each card has Approve/Deny buttons with confirmation pop-ups. Only shown if there are pending assignments.
- **Company Wall** *(conditional, toggleable)* — Monthly leaderboard ranking employees by completed projects. Top employee earns an "Employee of the Month" badge and highlighted profile for the following month. Only shown if the company has enough employees. Can be disabled in company settings.

#### Company Projects (`company_projects.dart`)
- **Employees** see only their own projects.
- **Managers/Officers** see all company projects, can manage them, review submissions, create new projects, assign members.

Project statuses: `Backlog` → `Pending Review` → `Done/Completed` or `Denied`

#### Company Team (`company_team.dart`)
- Displays all company members organized by category/role.
- Tapping a profile card allows direct messaging.
- Managers/Officers see an "Open Roles" button linking to `create_role.dart`.

### Key Flows

#### Project Creation and Assignment
1. Manager/Officer creates a project (`create_project.dart`).
2. Assigns members on `company_projects_assign.dart`.
3. Assigned users see the project on Company Home under "Assigned Projects" with Approve/Deny.
4. If approved, project moves to "My Projects" on Company Projects page.
5. A project group chat is automatically created with all assigned members.

#### Project Submission and Completion
1. Employee opens workspace (`company_workspace.dart`) from their project card.
2. Uploads/writes deliverables and submits for review.
3. Project status → `Pending Review`. Manager/Officer reviews on `company_home_queue.dart`.
4. If denied: manager writes feedback; employee can resubmit anytime.
5. If approved: status → `Done/Completed`. Added to employee's portfolio and company's completed projects.

#### Open Role Creation and Application
1. Manager/Officer creates a role on `create_role.dart` (title, description, skill requirements with levels).
2. Skills can be in-app (verifiable) or external (unverifiable).
3. Role becomes visible on `discover.dart` / `discover_hiring.dart` with skill-based recommendation matching.
4. User taps role → pop-up with details → taps Apply → `application.dart` (cover letter, file attachments).
5. Manager/Officer reviews applications on Company Team page. Can view applicant's profile, portfolio, skills, and completed projects.
6. If approved: applicant joins company as employee. If declined: application rejected.

## Matching and Recommendation System

### Open Role Discovery
- Users are recommended open roles on the Discover page based on the skills they are currently learning or have already completed, and their expertise level for those skills.
- This is the primary competitive advantage for companies using the platform — roles reach users who are actually qualified.

### Applicant Recommendations (Planned)
- When companies receive applications, an AI-powered filter will rank and recommend the best applicants for a given role based on their in-app profiles, skill levels, project history, and other signals.
- This solves the problem of companies being overwhelmed with unqualified applications.

### Future Enhancements to Matching
- Factor in **project complexity** (not just skill level).
- Factor in **recency** — a user who completed advanced projects recently is a stronger signal than someone who hit the same level years ago and hasn't been active.
- Factor in **pay expectations** once payment features are live.
